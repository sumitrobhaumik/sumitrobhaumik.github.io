Easy Tutorial for nnn

I recently tried to switch terminal file managers; from ranger to nnn. This is mainly because of two reasons.

  1. nnn is much faster than ranger. This difference is specially noticable when using them on a Raspberry Pi.
  2. Scripting is much easy in nnn

Having said that, I do like ranger in one regard; quick previews that work out of the box. Also, the application looks more like a traditional GUI file manager.

nnn, on the other hand, has a more spartan appearance; only one pane compared to ranger's three-pane interface. (Which can be changed with a little tweaking).

This guide should help you easily transition from ranger to nnn. Navigation is still the same, using the all too familiar VIM bindings. I'll still suggest to look at the keybindings page with ?. It's small and easy to look at, unlike ranger's 1000 line keybinding list.

I'm putting a list of actions that I commonly use and their corresponding key.

Action list

  1. Select and deselect - space

  2. Bulk rename current directory - r

    • This will open a file in your preferred editor
    • You can choose to edit the filenames you want to
    • You can also selectively chose to rename files using space

  3. Rename current file - ^r

  4. Edit current file in preferred editor - e
  5. Delete current file - x
  6. Exit at current directory - !
  7. Start shell - ]
  8. Sort - t followed by how you want to sort
  9. Directory size - t, followed by d
  10. Navigate as you type - ^n

  11. Add remote mount - c

    • Instead of typing user@IP, create a shortcut in ~/.ssh/config
    • Type the shortcut while mounting

  12. Access all remote mounts

    • Best to use bookmarks
    • All mounts are located at $HOME/.config/nnn/mounts
    • Bind key to the above location

  13. Disable wrap around while scrolling a list of items. It's also called edge rollover

    • Pass runtime argument -R
    • No environment variable available for this till now

Configuring nnn

Most of the configuration is done via environmental variables. I personally prefer config files since they are more portable across different installs. So I made my own implementation of a config file

All metadata of nnn resides in $HOME/.config/nnn/. So, I made a config file in the above mentioned location, put all the environment variables there, and sourced the file through my .zshrc/.bashrc file.

Bookmarks

This is the best way to navigate. Put loads of bookmarks and you can navigate much faster. You can also check your list of bookmarks by typing ?.

To use bookmarks, use the following environment variable. 

export NNN_BMS="p:<path 1>;q:<path 2>;q:<path 3>;"

The bookmark key is b. So, to jump to <path 1>, press bp.
